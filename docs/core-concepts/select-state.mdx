# Select State

The `SelectState` object contains all the reactive data that describes the current condition of the select component.

## Accessing State

You can access the current state using the `getState()` method on the select instance.

<CodeBlock>
```typescript
const state = select.getState();
console.log(state.isOpen);
```
</CodeBlock>

## Subscribing to Changes

In framework wrappers, state changes are handled automatically. If you are using the core library directly, you can subscribe to state updates:

<CodeBlock>
```typescript
const unsubscribe = select.subscribe((state) => {
  console.log('New state:', state);
});

// Clean up when done
unsubscribe();
```
</CodeBlock>

## Controlled vs. Uncontrolled

Headless Select supports both controlled and uncontrolled modes for the selection value.

* **Uncontrolled**: Pass `defaultValue`. The component manages the selection state internally.
* **Controlled**: Pass `value` and handle `onChange`. You are responsible for updating the `value` prop.
